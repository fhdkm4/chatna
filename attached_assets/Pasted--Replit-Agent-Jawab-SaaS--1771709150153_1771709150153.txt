مرحباً Replit Agent،

أعمل على مشروع "جواب" (Jawab)، وهو منصة SaaS لخدمة العملاء عبر واتساب باستخدام WhatsApp Business API الرسمي. لقد قمت بتحليل المتطلبات ووضعت خطة تطوير شاملة في محادثاتي السابقة. أحتاج منك تنفيذ المهام التالية بالترتيب:

1. تحديث نظام التسجيل (Authentication)

· إضافة زر "التسجيل باستخدام فيسبوك" (Embedded Signup) باستخدام Meta OAuth في صفحتي /register و/login.
· تغيير حقل "إشارة حساب" إلى "كود الخصم (اختياري)".
· إضافة نص توضيحي أسفل النموذج: "نحن نستخدم WhatsApp Business API الرسمي لضمان أمان رقمك وعدم تعرضه للحظر".

2. إنشاء معالج الإعداد السريع (Setup Wizard) في /wizard

· بعد التسجيل، ينتقل المستخدم إلى معالج من 3 خطوات:
  · الخطوة 1: ربط حساب Meta Business (إذا لم يكن قد ربط أثناء التسجيل).
  · الخطوة 2: عرض قائمة بأرقام الواتساب المتاحة في حساب المستخدم، مع تنبيه: "بمجرد الربط، سيتم نقل إدارة الرقم إلى منصة جواب ولن يعمل تطبيق الواتساب على جوالك لهذا الرقم".
  · الخطوة 3: اقتراح قوالب ردود جاهزة (ترحيب، ساعات عمل) ليقوم النظام بتفعيلها فوراً.

3. تطوير نظام دعوات الموظفين (Invitation System)

· في صفحة "إدارة الفريق" (/team)، يمكن الأدمن إضافة موظف باستخدام البريد الإلكتروني والدور فقط (admin/agent/manager).
· إنشاء توكن فريد لكل دعوة وتخزينه في قاعدة البيانات.
· إرسال بريد إلكتروني للموظف يحتوي على رابط قبول الدعوة (باستخدام Nodemailer).
· إنشاء صفحة /accept-invitation تسمح للموظف بتعيين كلمة المرور الخاصة به.

4. تنفيذ نظام الصلاحيات (Permissions)

· Middleware للتحقق من الأدوار:
  · Admin: صلاحية كاملة.
  · Manager: يمكنه إدارة الموظفين (إضافة، تعديل) لكن لا يمكنه حذفهم أو الوصول للإعدادات المالية.
  · Agent: يرى فقط لوحة المحادثات ولا يصل لصفحات الإعدادات أو الفريق.
· بعد تسجيل دخول الموظف، يتم توجيهه تلقائياً إلى /chat-dashboard.

5. إضافة ميزة التعليقات الداخلية (Internal Notes)

· في نموذج المحادثة، إضافة حقل isInternalNote: Boolean.
· في واجهة المحادثة، زر "إضافة ملاحظة داخلية" يرسل رسالة لا تظهر للعميل.
· عرض هذه الملاحظات داخل المنصة فقط مع اسم كاتبها.

6. تحديث Webhook للتكامل المباشر مع Meta Cloud API

· إنشاء endpoint /webhook للتعامل مع GET (للتحقق) وPOST (لاستقبال الأحداث).
· التحقق من صحة الطلبات باستخدام x-hub-signature-256.
· عند استقبال رسالة، البحث عن المستأجر (tenant) المرتبط برقم الهاتف ومعالجتها.

7. تخزين آمن لـ Access Tokens

· استخدام تشفير AES-256-GCM مع crypto.pbkdf2Sync لتشفير التوكنات قبل تخزينها في قاعدة البيانات.
· فك التشفير فقط عند الحاجة لإرسال رسالة أو استدعاء API.

8. إضافة "تأخير بشري" (Human-like Delay)

· دالة تحسب وقت التأخير المناسب بناءً على طول الرسالة (سرعة كتابة 52 كلمة/دقيقة + وقت قراءة).
· إرسال إشارة "جاري الكتابة..." عبر WebSocket قبل التأخير.
· تطبيق Rate Limiting ذكي (لكل مستأجر ولكل محادثة) لتجنب تجاوز حدود ميتا.

9. مراقبة جودة الحساب (Quality Rating)

· مهمة مجدولة (cron job) كل 6 ساعات تستدعي API ميتا لمعرفة تصنيف الجودة لكل حساب.
· عرض التصنيف في لوحة التحكم وإرسال تنبيهات إذا أصبح التصنيف أحمر.

الاعتماديات: Node.js، Express، MongoDB/Mongoose، Socket.io، Nodemailer، Redis (لـ rate limiting)، Meta Graph API v23.0.

لدي تعليمات برمجية مفصلة في المحادثة السابقة (تاريخ 22 فبراير) يمكنك الرجوع إليها. أريد منك البدء بالمهام بالترتيب وتزويدي بتحديث بعد كل خطوة.
