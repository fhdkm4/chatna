أريد ترقية نظام إدارة الفريق ليصبح احترافيًا ومتكاملًا، بدون تغيير الهيكل الحالي، فقط تطويره وتحسينه.

⚠️ مهم:
	•	لا نغير منطق المحادثات.
	•	لا نغير نظام Multi-Tenant.
	•	فقط تطوير إدارة الأعضاء والصلاحيات.

⸻

1️⃣ إضافة مسمى وظيفي مستقل (Job Title)

في جدول users أضف:
	•	job_title VARCHAR NULL
	•	avatar_url TEXT NULL

⚠️ role يبقى كما هو (admin / member)

الفرق:
	•	role = صلاحيات
	•	job_title = مسمى وظيفي يظهر تحت الاسم

في صفحة إدارة الفريق:
	•	يظهر تحت الاسم مباشرة job_title
	•	إذا كان null يظهر “—”

⸻

2️⃣ إنشاء صفحة ملف شخصي حقيقية

أنشئ صفحة:

/team/:userId

تعرض:
	•	صورة كبيرة دائرية (avatar_url أو حرف أول الاسم)
	•	الاسم
	•	المسمى الوظيفي
	•	الدور (admin/member)
	•	الحالة (online/offline)
	•	عدد المحادثات المفتوحة
	•	عدد المحادثات المغلقة
	•	زر مراسلة داخلي

⸻

3️⃣ تعديل الصورة الشخصية

السماح برفع صورة:
	•	فقط إذا currentUser.id === profileUser.id
	•	حفظ الصورة في storage (Cloudinary أو storage الحالي)
	•	تحديث avatar_url

لا يسمح للـ admin بتغيير صورة غيره.

⸻

4️⃣ تعديل المسمى الوظيفي

يسمح فقط إذا:

currentUser.role === ‘admin’
	•	يظهر زر تعديل صغير بجانب job_title
	•	لا يسمح للعضو بتعديل مسماه

⸻

5️⃣ حماية Backend حقيقية

جميع العمليات التالية يجب التحقق منها في backend:
	•	تغيير role
	•	حذف عضو
	•	تعديل job_title
	•	رؤية أعضاء

ويجب دائماً:

WHERE tenant_id = currentTenantId

⚠️ لا تعتمد على الواجهة فقط.

⸻

6️⃣ منع سيناريوهات خطيرة

أضف قيود:
	•	منع حذف آخر admin
	•	منع admin من حذف نفسه
	•	منع تغيير role لآخر admin إلى member

⸻

7️⃣ تحسين صفحة إدارة الفريق

أضف:
	•	حقل بحث بالاسم
	•	فلتر حسب الدور (الكل / مدير / موظف)
	•	ترتيب حسب النشاط
	•	عداد:
	•	المحادثات المفتوحة
	•	المحادثات المغلقة

⸻

8️⃣ إحصائيات أداء بسيطة لكل عضو

في Card العضو أضف:
	•	عدد المحادثات المفتوحة
	•	عدد المحادثات المغلقة
	•	عدد المحادثات المحولة من AI
	•	متوسط وقت الرد

لا نحتاج لوحة تحليل متقدمة، فقط أرقام واضحة.

⸻

9️⃣ تجربة مستخدم أفضل

عند الضغط على Card العضو:
ينتقل إلى صفحة الملف الشخصي.

لا نريد Modals، نريد صفحة فعلية.

⸻

🔟 لا نضيف تعقيد

لا نضيف:
	•	أدوار إضافية
	•	نظام صلاحيات معقد
	•	Departments حالياً

فقط:
admin / member
بشكل احترافي ومؤمن.

⸻

نفّذ بالترتيب:
	1.	Migration للـ DB
	2.	صفحة الملف الشخصي
	3.	رفع الصورة
	4.	حماية Backend
	5.	تحسين صفحة الفريق

وأبلغني عند الانتهاء من كل مرحلة.
