أحتاج تنفيذ تحديث مهم لتحويل النظام إلى SaaS متعدد المستأجرين بهوية AI مخصصة لكل شركة.

المطلوب تنفيذه بالكامل:

1️⃣ تحديث قاعدة البيانات (Multi-Tenant AI Identity)

أضف الحقول التالية إلى جدول tenants:
	•	businessDescription TEXT
	•	businessType VARCHAR
	•	contactPhone VARCHAR
	•	website VARCHAR
	•	workingHours JSONB
	•	address TEXT
	•	aiTone VARCHAR DEFAULT ‘friendly’
	•	welcomeMessage TEXT
	•	offHoursMessage TEXT
	•	aiPersonalityInstructions TEXT
	•	defaultEscalationMessage TEXT
	•	languagePreference VARCHAR DEFAULT ‘auto’

نفّذ migration آمنة بدون فقد بيانات.

⸻

2️⃣ إضافة صفحة “هوية الشركة” في الإعدادات

أنشئ قسم جديد داخل Settings باسم:

“هوية الشركة”

يحتوي على الحقول التالية (RTL + Dark Theme):
	•	اسم الشركة
	•	وصف مختصر
	•	مجال العمل (Dropdown: سفر وسياحة، تجارة إلكترونية، مطاعم، عقارات، صحة، تعليم، أخرى)
	•	رقم التواصل
	•	الموقع الإلكتروني
	•	ساعات العمل
	•	العنوان
	•	نبرة AI (formal / friendly / concise)
	•	رسالة ترحيب
	•	رسالة خارج أوقات العمل

مع:
	•	حفظ تلقائي
	•	زر حفظ
	•	التحقق من الحقول المطلوبة

⸻

3️⃣ بناء System Prompt ديناميكي لكل Tenant

أنشئ Prompt Builder احترافي بطبقات:

Structure:
	1.	BASE_SYSTEM
	2.	TENANT_IDENTITY
	3.	STYLE_LAYER
	4.	BUSINESS_INFO
	5.	STRICT_RULES
	6.	KNOWLEDGE_BASE
	7.	CONVERSATION_CONTEXT

يتم بناء الـ system prompt ديناميكياً باستخدام بيانات tenant.

قواعد إلزامية داخل الـ prompt:
	•	المساعد يمثل فقط اسم tenant.name
	•	ممنوع ذكر أي شركة أخرى
	•	الرد بنفس لغة العميل
	•	عند عدم معرفة الإجابة → تحويل لفريق [اسم الشركة]
	•	الرد أقل من 150 كلمة
	•	لا اختلاق معلومات

يجب أن يُستخدم نفس الـ prompt builder في:
	•	الرد الحقيقي
	•	ميزة المعاينة

⸻

4️⃣ عزل كامل بين الشركات (Data Isolation)

تأكد من:
	•	كل استعلام يحتوي WHERE tenant_id = ?
	•	قاعدة المعرفة مربوطة بـ tenant_id
	•	أي Vector DB تستخدم namespace أو metadata filter لكل tenant
	•	لا يوجد أي احتمال لتسريب بيانات بين الشركات

⸻

5️⃣ إنشاء Onboarding Wizard (3 خطوات)

عند تسجيل شركة جديدة لأول مرة:

Step 1: عرفنا عن شركتك
	•	اسم الشركة (required)
	•	وصف مختصر (required)
	•	مجال العمل (required)
	•	رقم التواصل
	•	الموقع

Step 2: خصص مساعدك الذكي
	•	اختيار النبرة (مع مثال لكل خيار)
	•	رسالة ترحيب (اقتراح تلقائي حسب المجال)
	•	ساعات العمل
	•	رسالة خارج الدوام

Step 3: أضف معلوماتك

عرض اقتراحات حسب المجال:
	•	سفر: الوجهات، الباقات، سياسة الإلغاء
	•	متجر: المنتجات، الاسترجاع، الدفع
	•	مطاعم: القائمة، التوصيل
	•	عقارات: العروض والأسعار

حقل نص كبير لإضافة معلومات مباشرة
زر “إضافة لاحقاً”

بعد الإكمال:
توجيه للداشبورد مع رسالة:
🎉 شركتك جاهزة! جرّب إرسال رسالة واتساب

⸻

6️⃣ إضافة ميزة “اختبار المساعد الذكي”

داخل صفحة الإعدادات:
	•	حقل يحاكي رسالة عميل
	•	زر “اختبار الرد”
	•	عرض رد AI الفعلي كما سيظهر في واتساب
	•	يستخدم نفس endpoint الحقيقي ونفس الـ prompt builder

⸻

7️⃣ متطلبات أساسية
	•	Dark Theme
	•	RTL
	•	تصميم متناسق
	•	Clean Code
	•	فصل منطق prompt builder في ملف مستقل
	•	كتابة كود قابل للتوسع

نفّذ كل شيء باحترافية وبهيكلية واضحة وقابلة للتطوير مستقبلاً.
